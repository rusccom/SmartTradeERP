# Региональные требования

Эта папка не является активной спецификацией проекта и не должна использоваться как источник обязательных задач при обычной разработке.

Содержимое здесь - справочная информация на будущее для оценки региональной пригодности ERP-системы. Не менять код, API, схему БД, UI или бизнес-логику на основании этих материалов, если пользователь явно не попросил реализовать конкретное региональное требование.

Если задача пользователя не связана с региональным compliance, эту папку не просматривать.
